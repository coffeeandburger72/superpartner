# Interests Builder

You are generating a structured `interests.md` profile file from the interests analysis results that were produced by the interests analyzer in the previous step. The analysis results are already held in context — do not re-read any source files.

Your job is to transform the analyzer's structured output into a clean, readable profile document and write it to disk using the Write tool.

---

## Instructions

### Step 1: Identify the Partner

From the analysis context, determine:
- The partner's name (as used throughout the chat)
- The partner's slug (lowercase, hyphen-separated version of the name — e.g., "jane-doe")
- Today's date in `YYYY-MM-DD` format

The output file path will be: `partners/<slug>/interests.md`

Create the directory if needed before writing the file.

### Step 2: Map Analyzer Output to Profile Sections

The analyzer organized findings into categories (Food & Drinks, Travel, Entertainment, Shopping & Brands, Hobbies & Activities, Pets). Map each category into the corresponding section of the output structure below.

**Tagging rules — apply to every item:**
- `(explicit)` — she directly stated a preference. Always include the original-language quote.
- `(inferred)` — a behavioral or conversational pattern implies a preference. Describe the evidence briefly.
- `(photo)` — extracted from a photo she shared. Describe what was visible.
- If an item has multiple evidence types, use the strongest tag and note additional evidence in the description.

**Ordering within each section:**
1. `(explicit)` items first
2. `(photo)` items second
3. `(inferred)` items last

**Language:** Keep all quotes and item names in the detected language of the chat. Do not translate original-language quotes.

**Wishlist items** are high-value for gift recommendations. Flag any item she expressed desire for but noted as too expensive, was "thinking about," or kept coming back to — include a note that it is a gift candidate.

### Step 3: Write the File

Use the Write tool to create `partners/<slug>/interests.md` with exactly the structure shown below. Populate every section with findings from the analysis. If a section has no evidence, write `- (nothing detected)` rather than omitting the section.

---

---

## Quality Rules

- Every item must have an evidence tag: `(explicit)`, `(inferred)`, or `(photo)`. No untagged items.
- Quotes must be reproduced exactly as they appear in the analyzer output — do not paraphrase or translate a quote labeled `(explicit)`.
- Negative preferences (dislikes, avoids) are as valuable as positive ones — include them.
- Consolidate duplicate entries: if the same item appears in multiple evidence types, merge into one entry using the strongest tag and listing all evidence.
- Wishlist items flagged by the analyzer must appear in the **Wishlist Items** section with the ⚠️ HIGH VALUE marker. These are the most actionable items for gift and date planning.
- If the analyzer flagged an item as a wishlist candidate (too expensive, mentioned multiple times, "thinking about it"), carry that flag into the profile and add the ⚠️ HIGH VALUE marker.
- Do not invent items not present in the analyzer output.
- If the analyzer noted uncertainty, preserve that uncertainty — use `(inferred)` and describe the ambiguity.
- Write the file in the same language as the chat (typically Traditional Chinese / Cantonese with English mixed in). Section headers remain in English.
