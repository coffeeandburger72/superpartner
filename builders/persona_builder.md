# Persona Builder

This file instructs Claude how to generate the final `persona.md` profile file from persona analysis results. When SKILL.md reads this file during the build phase, follow every instruction below to write the persona profile.

---

## Input

You have the following data held in context from the analyzer step:

- **Persona analysis results** — structured output from `analyzers/persona_analyzer.md`, covering all 6 layers (Core Traits, Identity, Communication Style, Emotional Patterns, Values & Boundaries, Attachment Orientation)
- **Partner metadata** — slug, display name, message count, date range of the chat export
- **Today's date** — used in the file header

---

## Task

Write the file `partners/<slug>/persona.md` using the Write tool.

Replace `<slug>` with the actual partner slug (e.g., `jane-doe`).

---

## Section-by-Section Instructions

### Header

- `<Name>` — use the partner's display name exactly as stored (e.g., `Jane Doe`)
- `<date>` — today's date in `YYYY-MM-DD` format
- `<message_count>` — total message count from the parsed export
- `<date_range>` — formatted as `DD MMM YYYY – DD MMM YYYY` (e.g., `01 Jan 2024 – 31 Mar 2025`)

---

### Core Traits (Layer 0)

Pull directly from the Layer 0 analysis results. Each bullet must be a behavioral rule in the form "situation → behavior pattern" — not a vague adjective.

**Formatting rules:**
- Each bullet is a single sentence describing what she does when X happens
- Append the confidence tag in parentheses at the end: `(strong)`, `(moderate)`, or `(weak)`
- After each bullet, add an indented sub-bullet with a supporting quote or pattern description as evidence

**Good example:**
```
- When the user mentions a stressful day at work, she sends an encouraging message and follows up the next morning to ask if things improved. (strong)
  - Evidence: "加油呀，你得架 💪" followed by "今日好啲未？" the next day — seen across 6+ separate conversations
```

**Bad example:**
```
- She is caring and supportive.
```

Include a minimum of 3 traits and a maximum of 8. Select the most frequently evidenced ones.

---

### Identity (Layer 1)

Populate the table using Layer 1 analysis results.

**Column rules:**
- `Field` — use the exact field names from the template
- `Value` — fill in the extracted value; if unknown write `(insufficient data)`
- `Confidence` — use `—` for fields that are factual (Name, Languages); use `strong`, `moderate`, or `weak` for inferred fields (Age, Job/Industry, MBTI)

**MBTI formatting:** If MBTI was directly stated in chat, write the type and append `(confirmed)`. If inferred, write the type and append `(inferred)`. Example: `ENFP (inferred)`.

**Languages formatting:** List all languages she uses in messages, with a note on code-switching if observed. Example: `Cantonese (primary), English (work topics, loanwords)`.

---

### Communication Style (Layer 2)

#### Messaging Habits

Write 2-4 bullet points covering:
- Typical message length (short bursts vs long paragraphs)
- Frequency and response time patterns
- Whether she double-texts or sends multiple messages before a reply
- Any notable patterns (e.g., goes quiet during work hours, active late at night)

#### Emoji & Sticker Profile

- `Top emoji` — list the top 5 emoji from the analysis with approximate frequency, e.g.: `😂 (very frequent), 🥺 (frequent), 💀 (moderate), 😭 (moderate), 🙈 (occasional)`
- `Sticker style` — summarize the dominant tone and top characters/IPs from sticker analysis. If no stickers were sampled, write `(no sticker data available)`

#### Tone by Context

Fill each time slot based on the Layer 2 tone shift analysis. If there is no evidence for a particular slot, write `(insufficient data)` — do not omit the line.

#### Example Messages

Include 3-5 quoted messages pulled directly from the Layer 2 "Example Messages" section of the analysis. Each quote goes inside a `>` blockquote on its own line.

**Rules:**
- Quotes must be in the original language — do not translate
- Cantonese stays Cantonese, mixed-language messages stay mixed
- Add a brief italicised label before each quote indicating the mood it demonstrates

**Formatting:**
```
*Happy/excited* —
> [quote here]

*Tired/stressed* —
> [quote here]

*Affectionate* —
> [quote here]
```

---

### Emotional Patterns (Layer 3)

#### What Excites Her

List 3-5 topics or situations that consistently trigger excitement, drawn from Layer 3 analysis. Each bullet must include evidence — either a quoted message or a description of the pattern.

#### What Frustrates Her

List 3-5 frustration triggers with evidence. Include signal indicators where relevant (e.g., shorter replies, topic avoidance, specific emoji).

#### How She Shows Affection

Write 2-4 bullets describing her affection patterns. Each bullet should include a concrete example or quote. Distinguish between direct verbal expression and indirect acts of care.

#### Love Language Ranking

List all 5 love languages ranked 1 (strongest) to 5 (weakest), based on Layer 3 analysis. For each:
- State the love language name
- Provide 1-2 sentences of supporting evidence
- If evidence is thin, append `(insufficient data — limited signal in text)`

---

### Values & Boundaries (Layer 4)

#### Takes Seriously

List topics where her tone demonstrably shifts to serious — fewer emoji, longer messages, more careful word choice. Each bullet: topic name, then a brief description of the tonal shift with an example.

#### Avoids / Sensitive Topics

List topics she consistently deflects, avoids, or does not engage with. Each bullet: topic name, then the avoidance pattern (e.g., "changes subject when X comes up", "gives one-word answers to questions about Y").

#### Life Priorities

List 3-5 priorities ranked by conversational energy — how much space they take up in her messages. Each bullet: priority name, then the evidence (what she talks about, complains about, plans around).

---

### Attachment Orientation (Layer 5)

#### Style

Write the classified orientation from the Layer 5 analysis. Use only these terms: `Secure`, `Anxious`, `Avoidant`, or `Mixed`. Do not use clinical language ("anxious attachment style") — frame as a behavioral orientation.

If insufficient data, write: `(insufficient data — need more chat history to assess attachment patterns)` and skip the subsections below.

#### Behavioral Signals

List 3-5 behavioral signals from the Layer 5 analysis, formatted identically to Core Traits:
- Each bullet is a situation → behavior pattern
- Each bullet has an indented evidence sub-bullet with an original-language quote
- Confidence tag on each bullet

**Good example:**
```
- When the user doesn't reply for 2+ hours, she sends follow-up messages checking in and her tone shifts from casual to worried. (strong)
  - Evidence: "你喺邊呀 😟" → "點解唔覆我" → "你係咪唔開心" — this escalation pattern appears across 6+ separate instances
```

**Bad example:**
```
- She has anxious attachment.
```

#### Activation Triggers

List 2-4 specific situations that activate attachment insecurity or withdrawal. Each bullet: the trigger situation, her behavioral response, and a brief evidence reference.

#### What This Means for You

This is the first advisory section in persona.md. Write exactly 1-2 sentences that translate her attachment orientation into practical guidance for the user. Ground every recommendation in the behavioral signals listed above.

**Good example:**
> She responds best when plans are confirmed early and changes are communicated proactively — ambiguity about logistics triggers her check-in patterns.

**Bad example:**
> She needs lots of reassurance because she has an anxious attachment style.

Rules:
- Never use psychological labels ("anxious attachment", "avoidant style")
- Never give generic relationship advice
- Connect directly to the specific signals in the section above

---

## Evidence Rules

Apply these rules to every section:

1. **Every bullet must have evidence.** Either a direct quote in `>` blockquote format, or a pattern description specific enough to be verified.
2. **Keep quotes in original language.** Never translate. Cantonese stays Cantonese, mixed-language messages stay mixed-language.
3. **Do not invent or extrapolate.** If the analysis output says `(insufficient data)`, carry that through to the profile — do not fill gaps with guesses.
4. **Confidence tags** — include them in Core Traits (Layer 0) and Identity table (Layer 1). They are optional but welcome in other sections for high-stakes claims.
5. **Sections with no data** — include the section header and write `(insufficient data)` as the only content. Never omit a section entirely.

---

## Quality Checklist

Before writing the file, verify:

- [ ] Header contains correct name, date, message count, and date range
- [ ] Every Core Trait bullet is a behavioral rule (situation → behavior), not a vague adjective
- [ ] Every Core Trait bullet has a supporting evidence sub-bullet
- [ ] Identity table has all 5 rows filled (no missing rows)
- [ ] Example Messages section contains 3-5 actual quotes in original language, formatted as `>` blockquotes
- [ ] Love Language Ranking lists all 5 languages in order with evidence
- [ ] No section is omitted — sections with no data say `(insufficient data)`
- [ ] No translations of partner quotes — original language only
- [ ] Attachment Orientation section is present (even if marked insufficient data)
- [ ] No psychological labels appear anywhere in Layer 5 — only behavioral descriptions
- [ ] "What This Means for You" is 1-2 sentences max, grounded in observed signals
- [ ] Behavioral Signals use the same situation → behavior format as Core Traits

Once the checklist passes, write the file using the Write tool to `partners/<slug>/persona.md`.
