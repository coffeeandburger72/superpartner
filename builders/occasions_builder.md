# Occasions Builder

You are generating a structured `occasions.md` profile file from the occasions analysis results already held in context from the occasions analyzer step.

Your job is to transform the raw analysis into a clean, scannable reference document that can be consulted at a glance — especially when planning gifts, surprises, or scheduling time together.

---

## Your Task

Using the occasions analysis results in context, generate the file `partners/<slug>/occasions.md` using the Write tool.

Replace `<slug>` with the partner's slug (lowercased, hyphenated name, e.g., `jane-doe`).

---

## Output Structure

Write the file with exactly this structure:

```markdown
# <Name> — Important Dates

> Generated by superpartner on <today's date in YYYY-MM-DD format>.

## Fixed Dates

| Occasion | Date | Confidence | Source |
|----------|------|------------|--------|
| Birthday | [date] | [strong/moderate/weak] | "[quote or context]" |
| Anniversary | [date] | [strong/moderate/weak] | "[quote or context]" |
| [Other fixed date] | [date] | [strong/moderate/weak] | "[quote or context]" |

## Recurring Events

| Event | Frequency | Day/Time | Notes |
|-------|-----------|----------|-------|
| [e.g., Dim sum] | Weekly | Sunday morning | "with family" |
| [e.g., Gym] | 3x/week | Weekday evenings | ... |

## Upcoming Events

| Event | Date (approx) | Details |
|-------|---------------|---------|
| [e.g., Taiwan trip] | April 2026 | "with friend, 5 days" |

## People She Cares About

| Name | Relationship | Notable Dates | Notes |
|------|-------------|---------------|-------|
| [e.g., Mom] | Mother | — | "goes to Japan together" |

## Gift Calendar (Next 90 Days)

Based on fixed dates and upcoming events, the next occasions requiring attention:

1. [occasion] — [date] — [suggested lead time for preparation]
2. ...
```

## Rules

### General

- Use tables for all structured data — easier to scan at a glance.
- Always include the confidence level (strong / moderate / weak) for every date in Fixed Dates.
- Keep all quotes in their original language — do not translate Cantonese, Mandarin, or other languages.
- Do not fabricate or infer dates beyond what the analysis provides. If something was marked uncertain in the analysis, carry that uncertainty into the output.

### Birthday Handling

- If birthday is known, include it in the Fixed Dates table as normal.
- If birthday is UNKNOWN (not found in analysis), add this notice at the very top of the file, directly below the `> Generated by...` line and before any section heading:

  ```
  > **Birthday: UNKNOWN — ask directly or watch for hints**
  ```

  Also include a row in the Fixed Dates table:

  | Birthday | UNKNOWN | — | No evidence found in chat |

### Fixed Dates Section

Include all of the following if found in the analysis:
- Partner's birthday
- Relationship anniversary / start date
- Family members' birthdays (Mom, Dad, siblings, etc.)
- Close friends' significant events (weddings, engagements, etc.)

Omit rows where no evidence exists — do not leave empty placeholder rows.

### Recurring Events Section

Include only events that show a genuine repeating pattern (weekly, monthly, seasonal). Single instances do not qualify. Use the Day/Time column to capture specifics like "Sunday morning" or "Wednesday evenings." If timing is vague, write "varies."

### Upcoming Events Section

- Include only future-facing events relative to today's date.
- Use absolute dates (e.g., "April 2026", "2026-05-10") — never relative expressions like "next month."
- Append **(approx)** to the date if it was marked approximate in the analysis.
- If there are no upcoming events identified, write a single row: `| No upcoming events identified | — | — |`

### People She Cares About Section

- Include family members, close friends, and any other people who appear meaningfully in the analysis.
- The "Notable Dates" column should list birthdays or significant dates if known; use `—` otherwise.
- Use the "Notes" column for brief, useful context (relationship dynamics, shared activities, location, etc.).
- Prioritize people mentioned frequently or with emotional significance.

### Gift Calendar (Next 90 Days)

The Gift Calendar is a convenience view — a chronological list of occasions in the 90-day window from today that require planning or action.

Rules for this section:
- Include fixed dates (birthdays, anniversary), upcoming events (trips, concerts, social events), and any other occasion where a gift, gesture, or coordination is appropriate.
- List items in chronological order.
- For each item, suggest a lead time for preparation (e.g., "order 2 weeks ahead", "book restaurant 1 week ahead", "buy gift 3–5 days before").
- If no occasions fall within the 90-day window, write: `No occasions identified in the next 90 days.`
- If the birthday is unknown, include as item 1 regardless: `Birthday (date unknown) — watch for hints or ask directly`

## Quality Checklist

Before writing the file, verify:

- [ ] The slug in the file path matches the partner's name
- [ ] Today's date is used for the "Generated by" line and all 90-day calculations
- [ ] Birthday UNKNOWN notice is present at the top if birthday was not found
- [ ] All dates are absolute — no relative expressions remain
- [ ] Confidence levels are present on every row in Fixed Dates
- [ ] All quotes are in original language
- [ ] Gift Calendar covers a 90-day window from today and is in chronological order
- [ ] No placeholder rows exist for sections with no data — omit or use a clear "none identified" row instead

---

## Write the File

Once you have verified the checklist, use the Write tool to create the file at:

```
partners/<slug>/occasions.md
```

Do not print the file contents to the conversation after writing — just confirm the file was written and state the path.
